# QA manual — checklist pós-sessão 2026-06-15

Bateria de smoke tests manuais pra rodar em prod (https://bam-araujo.github.io/app-financas-bam-evellyn/) e validar que tudo da sessão está em pé. Tempo total estimado: ~20min.

Marca cada item ✅ ou ❌ conforme passa/falha. Se falhar, copia o erro do console (F12) e me chama.

## Setup (1min)

- [ ] Abrir app em prod, fazer login.
- [ ] Confirmar que header mostra nome + foto do usuário logado.
- [ ] Console (F12) limpo de erros vermelhos no boot.

## Receitas — série (5min)

- [ ] Em Receitas, clicar `+ Nova`. Form abre.
- [ ] Repetição: **Recorrente**. Pessoa: você. Tipo: Salário. Valor: 100. Salvar.
- [ ] Trocar competência pra mês +1, +2, +3, +6. Receita aparece em todos. ✅ = parcelas auto-estendidas funcionando.
- [ ] Voltar pra competência atual. Editar a receita. **Dialog "Esta + futuras / Só esta"** deve aparecer.
- [ ] Escolher "Esta + futuras", mudar valor pra 200. Conferir +1, +2: valor 200. Conferir mês passado (se tiver — vai depender do seed): permanece original. ✅ = forward propaga corretamente.
- [ ] Editar a receita do mês atual de novo, escolher Repetição "Única". Confirmar conversão. Mês +1 deve sumir, mês atual continua. ✅ = conversão série→única.
- [ ] Limpar: deletar a receita criada (escolher "esta + futuras" se aparecer dialog).

## A6 — Previsão de caixa (3min)

- [ ] Na Home, card "Previsão de caixa — 6 meses" aparece.
- [ ] Toggle "Conta" / "Patrimônio total" funciona. "Patrimônio total" desloca a linha pra cima pela soma dos investimentos.
- [ ] Editar "Saldo em conta hoje" — input aceita digitar, "Salvar" persiste, valor reaparece ao recarregar a página.
- [ ] Mudar filtro de pessoa (casal → Bam → Evellyn). Card recalcula. Saldo inicial muda (cada pessoa tem o seu em localStorage). ✅ = isolamento por pessoa.
- [ ] Tabela embaixo mostra 6 meses, entradas/saídas/saldo. Saldo negativo aparece em vermelho.

## Importação — Itaú (2min)

- [ ] Importar > Banco: "Detectar automaticamente". Sobe um PDF Itaú.
- [ ] Detecção identifica Itaú. Tela de review mostra fatura: titular, vencimento, total.
- [ ] Hint logo abaixo: "Pagador e dono pré-preenchidos como **<Nome>** (titular da fatura)". ✅ = prefill por titular.
- [ ] Campos com labels visíveis (DATA, DESCRIÇÃO, VALOR, CATEGORIA, TIPO, PAGADOR, DONO, REPETIÇÃO).
- [ ] Linhas com parcela detectada: badge "parser detectou N/M" + repetição = parcelado + nº de parcelas preenchido.
- [ ] Salvar com 1-2 linhas selecionadas. Mensagem "X lançamentos criados nesta fatura" (sem inflar com parcelas futuras).
- [ ] Conferir as linhas em Despesas — dono está correto (= titular da fatura).

## Importação — Nubank (2min)

- [ ] Mesmo fluxo com PDF Nubank.
- [ ] Auto-detect identifica Nubank.
- [ ] Transações da seção "TRANSAÇÕES DE..." aparecem todas. Pagamentos negativos NÃO aparecem.
- [ ] Linhas IOF aparecem como tx separadas (descrições curtas, valores pequenos).
- [ ] Salvar. Conferir em Despesas.

## Importação — Mercado Pago (2min)

- [ ] Mesmo fluxo com PDF MP.
- [ ] Auto-detect identifica Mercado Pago (não atropela como Itaú).
- [ ] "Pagamento da fatura de abril/2026" **não aparece** na lista.
- [ ] "Tarifa de uso do crédito emergencial" **aparece**.
- [ ] Linhas com "Parcela N de M" no PDF vêm com repetição=parcelado + parcelas calculadas.
- [ ] Salvar. Conferir.

## PWA — auto-update (depende do timing)

- [ ] **Próximo deploy que rolar:** abrir o PWA instalado uma vez. Você vê a versão antiga rapidamente, depois reload silencioso → versão nova aparece sem precisar fechar e abrir.
- [ ] Se ainda exigir 2-3 ciclos, NÃO é regressão — significa que o SW antigo (de antes do `b81b34a`) ainda está em cache no seu device. Próximo deploy depois desse fica OK.

## Acerto + cache no-store (1min)

- [ ] Em Despesas, editar uma despesa CONJUNTA (descrição, valor, qualquer coisa). Salvar.
- [ ] Ir pra Acerto na mesma competência.
- [ ] A despesa editada aparece com **valor/descrição NOVOS** imediatamente, sem precisar F5. ✅ = `cache: 'no-store'` funcionando.

## Quando algo falha

1. Copiar erro do console (F12 → Console → erro vermelho).
2. Tirar screenshot da tela.
3. Anotar o passo exato que quebrou.
4. Próxima sessão: passa esses 3 itens pro agente. Ele consegue diagnosticar em ~5min.
